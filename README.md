### 🧠 About Me

I'm an **AI Engineer at Volvo Group Digital & IT**, building **production-grade AI applications** — from analytical ML pipelines to generative AI agents that handle real traffic every day. My bias is strong: ship things that actually run, survive, and scale beyond the notebook.

- 🤖 **Current focus:** Agentic systems, RAG (naive → agentic), MCPs, text2sql, and enterprise chatbots
- ☁️ **Cloud-first:** Azure + Databricks, end-to-end — *real production systems, not prototypes*
- 🏗️ **Full lifecycle:** Training → serving → monitoring → integration with on-prem enterprise systems
- ⚙️ **Daily stack:** Python (FastAPI), React, Azure OpenAI, Azure AI Search, Service Bus, MLflow
- 🎓 **Background:** B.Sc. Software Engineering (Kharazmi University) · M.Sc. Artificial Intelligence (Stockholm University)
- 💬 **Ask me about:** agentic RAG, productionizing LLM apps, Databricks MLOps, Azure architecture, automation workflows

### 🚀 Featured Production Systems

> Real systems deployed at Volvo Group — handling production traffic every day.

    <td width="50%" valign="top">

#### 💬 Group Finance Chatbot

An advanced enterprise **Q&A assistant** for financial reporting documentation — with live connections to data sources and intelligent, multi-strategy retrieval.

**Highlights**
- Query router dispatches to the right retrieval strategy — **RAG**, **text2sql**, or **text2pandas**
- Full conversation management with session persistence
- End-to-end secured: Microsoft SSO + Graph API integration
- Cloud-deployed, end-to-end monitored

**Tech stack**

`Azure OpenAI` · `Azure AI Search` · `FastAPI` · `React` · `Redis` · `MongoDB` · `Microsoft SSO`

</td>
<td width="50%" valign="top">

#### 🔧 Warranty AI

A large-scale **claim evaluation platform** combining rules-based checks, ML models, and GenAI-powered agents. Processes **thousands of claims daily**.

**Highlights**
- Pre-compute feature pipelines on Databricks → Unity Catalog + Azure SQL
- Model training, serving & monitoring with MLflow (end-to-end MLOps)
- Inference worker on Azure Functions, orchestrated via Service Bus
- Integrated with on-prem claim-handling systems

**Tech stack**

`Databricks` · `MLflow` · `Azure Functions` · `Service Bus` · `PySpark` · `Azure OpenAI`

<sub>Built from day 1 in a ~20-engineer monorepo.</sub>
